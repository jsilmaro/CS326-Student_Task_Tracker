# CI Notes

## Continuous Integration Setup (Simulated)

For this project, CI is simulated using GitHub Pull Requests and manual testing.

## Workflow:
- Feature branches are created for each change
- Pull Requests are used for code review
- Tests are run locally before merging
- Only tested code is merged into main branch

## Benefits:
- Ensures code quality
- Reduces bugs in main branch
- Encourages structured workflow
